## Maintenance Notes

- Inline intent manifests remain supported for v0.7 compatibility.
- API and MCP reject requests that provide both `manifest` and `policy_pack_path`.
- `parameters_schema` is validated as a JSON object and, in v0.8.1, evaluated at runtime with a small deterministic subset: `type`, `required`, `properties`, `enum`, string length bounds, numeric bounds, and array `items`.
- Output guardrail policy modes in v0.8.2 let API/MCP/Python callers choose `allow`, `review`, or `block` behavior for unknown, unapproved, and secret terms while preserving strict default blocking.
- Guardrail Review Packet in v0.8.3 adds deterministic grouped review evidence for unknown, unapproved, and secret output terms without introducing persistence, rewriting, or external integrations.
- `OMNIGLYPH_POLICY_PACK_ROOT` mirrors the Lexicon Pack root restriction for API and MCP only.
- Policy Packs are local files; no database ingestion or command execution is introduced.
