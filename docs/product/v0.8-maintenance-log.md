## Maintenance Notes

- Inline intent manifests remain supported for v0.7 compatibility.
- API and MCP reject requests that provide both `manifest` and `policy_pack_path`.
- `parameters_schema` uses a documented lightweight subset (`type`, `required`, `properties`, `enum`, string length bounds, numeric bounds, and array `items`). Supported keywords are meta-validated before value checks; malformed keyword values fail closed (inline → `invalid_manifest`; Policy Pack → validate/load failure). Unknown keywords remain ignored. Schema and value traversal are iterative so deep or cyclic schemas return findings instead of `RecursionError`.
- Intent-policy fail-closed hardening ensures invalid Policy Packs cannot load and invalid inline manifests return `decision: "block"` / `status: "invalid_manifest"` instead of allowing or raising.
- Output guardrail policy modes in v0.8.2 let API/MCP/Python callers choose `allow`, `review`, or `block` behavior for unknown, unapproved, and secret terms while preserving strict default blocking.
- Guardrail Review Packet in v0.8.3 adds deterministic grouped review evidence for unknown, unapproved, and secret output terms without introducing persistence, rewriting, or external integrations.
- Guardrail Review Packet Hardening in v0.8.4 adds direct `unapproved` and policy-`allow` branch coverage and consolidates action precedence into one runtime constant.
- Output Guardrail CLI in v0.8.5 exposes `enforce_grounded_output` as `omniglyph enforce-output` with repeated `--term`, optional `--policy`, and optional `--actor-id`.
- Source-batch closeout documentation records the current branch, package-version boundary, verification command, key paths, and registry-publication next steps.
- Online readiness hardening on 2026-07-07 adds CLI coverage for `validate-output`, `scan-language-input`, `scan-output-dlp`, and `audit-explain` while preserving the v0.8 `enforce-output` and `enforce-intent` syntax.
- `OMNIGLYPH_POLICY_PACK_ROOT` mirrors the Lexicon Pack root restriction for API and MCP only.
- Policy Packs are local files; no database ingestion or command execution is introduced.
