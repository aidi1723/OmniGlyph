## Operational Notes

- Run `scripts/release_check.sh` before any release, PR merge, PyPI upload, or MCP registry update.
- Use exact artifact filenames for TestPyPI/PyPI uploads. Do not upload with `dist/*`, because stale artifacts can be published accidentally.
- `scripts/artifact_audit.py` derives default artifact names from `pyproject.toml`; update the version there first when preparing a new release.
- `scripts/wheel_smoke_test.sh` installs the local wheel with `--no-deps` to validate package entry points and packaged files without relying on network access.
- For Python 3.10 release environments, install the `dev` extra so `tomli` is available for artifact audit TOML parsing.
- `scan_code_symbols` is retained as a deprecated MCP alias. Keep it until downstream MCP clients have migrated to `scan_unicode_security`.
